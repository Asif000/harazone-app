Design Decisions

  • Light theme by default for the entire detail page below the hero image. Chat bg #F5F2EF (warm off-white), AI bubbles #FFF with subtle border, input #EBE7E3.
  • Hero stays dark — it's a photo with gradient overlay. The dark-to-light transition happens at the POI info section. No jarring cut.
  • System theme support: These light values become the default. When device is in dark mode, swap to the current dark palette. Two color schemes, one layout.
  • AI Context Block is white card with primary-color left border + subtle shadow. Reads like editorial content, not a UI widget. Contains contextBlurb + whyNow + localTip.
  • Chat visual hierarchy: Light bg makes chat feel like a reading experience, not a dark terminal. User bubbles (indigo gradient) pop on light bg. AI bubbles are white cards with hairline border.
  • Pills are white with light border — feel tappable but not heavy. Input bar uses warm gray (#EBE7E3).
  • Bottom nav hidden when detail page is open. FAB persists.
  • Placeholder text: "Ask about this place..." (contextual, not generic).

Color Palette — Light Theme (Default)

  • Chat background: #F5F2EF — warm off-white (not cold/blue white)
  • AI bubbles / Context block / Pills: #FFFFFF with rgba(0,0,0,0.06) border
  • Chat input: #EBE7E3 — slightly darker warm gray
  • "Why now" inset: #FAF7F4 — barely-there warm tint
  • Primary text: #1A1412 — warm near-black (matches app's OnPrimary)
  • Secondary text: #7A6E64 — warm mid-gray
  • Muted text: #A89888 — placeholder, timestamps
  • Primary accent (on light): #C47A3E — slightly deeper than #E89A5E for contrast on white

System Theme Strategy

  • Default: Light — detail page uses light palette regardless of map (map always dark).
  • Dark mode override: When system dark mode is active, detail page swaps to current dark palette. Single isSystemInDarkTheme() check.
  • Implementation: Create DetailPageColorScheme with light/dark variants. Compose's MaterialTheme wrapper scoped to detail page only. Map + carousel keep dark theme always.
  • Tester release: Ship light-only first. Add dark mode toggle as fast follow. Simpler to test one theme well than two themes poorly.
