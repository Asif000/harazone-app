8. Simplified Card Actions + Pin States

All results shown — nothing filtered. Saved POIs are HIGHLIGHTED (glow + badge). Visited/closed are DIMMED (ghost, 50% opacity). Map card simplified to 3 actions: Save / Go / Details.

Decision: Never filter results — users perceive fewer results as low quality. Saved = highlighted (they matter). Visited/closed = dimmed (done/unavailable).
Key insight: Saved pins should stand out MORE not less. They're places the user actively cares about.

Design Decisions Captured

  • All results shown — NEVER filter results. Fewer results = perceived low quality. Saved and visited POIs appear in search alongside fresh ones.
  • Saved pins = HIGHLIGHTED — glow effect + ♡ badge. These are places the user cares about — they should stand out MORE.
  • Visited/closed = DIMMED — ghost pin at 50% opacity + ✓ badge. De-prioritized but visible for spatial awareness.
  • Card = info preview + ♡ only — no action buttons. Rich info display: image, name, vibes, rating, status, price, distance, AI snippet. One ♡ heart for quick save. Tap card body → detail page.
  • Detail page = action center — Save/Unsave, Go Now, Go Later, AI planning, image carousel, full narrative. All CTAs live where the AI agent has full context.
  • Saved places list = personal collection with Go Now / Go Later / Remove. Planning workflow lives here.
  • Quick ♡ during slideshow — tap heart without interrupting rotation. Like Instagram double-tap.
