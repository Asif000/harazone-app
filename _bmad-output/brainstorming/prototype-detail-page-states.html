State Flow & Anti-Clustering Rules

  • State 1 (just opened): Hero + POI info + shimmer context block + empty chat with hint text + pills. Send button disabled. User sees the place, waits ~1-2s for AI context.
  • State 1b (context loaded): Shimmer replaced by AI Context Block. Chat still empty with hint. Pills are the primary CTA — inviting first tap.
  • State 2 (conversation): After pill tap, natural conversation flow. POI cards appear INLINE but never back-to-back. Always separated by AI text.
  • Anti-clustering rule: AI text MUST appear between every POI card. Pattern: text → card → text → (user) → text → card → text. No card stacks. This is a prompt contract rule.
  • Single-POI responses: When AI mentions just one place, one card appears with surrounding text. No card rail needed.
  • Multi-POI responses: When AI recommends 3+ places, cards can appear in sequence BUT with a brief text line between each (e.g., "And for a different vibe:"). This was shown in the previous prototype for the "list" use case — acceptable clustering only when user explicitly asks for a list.
  • Pill state: Tapped pill gets active highlight. Pills update contextually — "Getting there" replaces "Budget tips" after dessert mention.
  • Empty state hint: "Tap a suggestion below or ask anything about this place" — centered, muted, disappears on first message.

Prompt Contract — Card Spacing Rules

  • Rule 1: Every POI mentioned by name in prose MUST have a corresponding card in the pois array.
  • Rule 2: When returning multiple POIs, include a brief text segment between each card (context, comparison, or transition).
  • Rule 3: Exception — if user explicitly asks "list all nearby X," cards may appear consecutively (max 3) with a text intro and text summary after.
  • Rule 4: The tapped/focused POI always has a guaranteed card (belt-and-suspenders: prompt + client-side injection).
  • Client-side enforcement: If Gemini returns consecutive cards with no text between them, client inserts a minimal separator (e.g., thin divider line) to prevent visual clustering.
